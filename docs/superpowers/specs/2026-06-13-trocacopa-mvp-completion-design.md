## 2. Mobile-First Layout

All pages are designed mobile-first. Specific requirements:

- Touch targets ≥ 44px (shadcn Button already handles this)
- Bottom navigation bar on mobile (≤ md breakpoint) replacing hamburger menu
- Content padding accounts for bottom nav (`pb-20 md:pb-0`)
- Max content width `max-w-6xl` centered on desktop
- Grids start at 1 column mobile → expand on tablet/desktop
- Main app layout adds bottom nav as a fixed bar visible only on mobile

### Bottom Nav (mobile, ≤ md)
4 links: Painel (`/dashboard`), Álbum (`/album`), Repetidas (`/repetidas`), Trocas (`/trocas`)  
Active state: primary color on icon + label.

## 4. Payment — One-Time Purchase (Stripe)

### Model
- **Free:** landing `/`, sign-in, sign-up, public profile `/u/[slug]`
- **Paid:** all authenticated routes (`/dashboard`, `/album`, `/repetidas`, `/trocas`, `/perfil`)
- **Price:** R$19,90 (configurable via `PRICE_CENTS` env var = `1990`)
- **Method:** Stripe Checkout hosted page — Pix + cartão de crédito/débito

### DB — `purchase` table
```
id          serial PK
userId      text FK → user.id (unique — one purchase per user)
status      text default 'pending' ('pending' | 'paid')
stripeSessionId text nullable
paidAt      timestamp nullable
createdAt   timestamp default now()
```

### Flow
```
signup → app layout checks purchase status
  → status ≠ 'paid' → redirect /comprar
  → /comprar shows price + features → click "Comprar"
  → POST /api/checkout → Stripe Checkout Session → redirect to Stripe
  → user pays (Pix or card) → Stripe calls webhook
  → POST /api/webhooks/stripe → mark purchase as 'paid'
  → user redirected to /dashboard?pagamento=sucesso
```

### New env vars required
```
STRIPE_SECRET_KEY=sk_live_...
STRIPE_WEBHOOK_SECRET=whsec_...
PRICE_CENTS=1990
```

### Gate
`app/(app)/layout.tsx` calls `getPurchaseStatus(userId)` before rendering. If not `'paid'`, redirects to `/comprar`.

## 5. Missing Pages

### `/repetidas` (server component, authenticated)
- Lists all stickers with count > 1, grouped by team/section
- Header: total duplicate codes + total extra copies
- ShareButton: copies plain-text list to clipboard
- Teams with zero duplicates hidden

### `/trocas` (server component, authenticated)
- Calls `getOtherCollectors` + `computeMatch` per collector
- Sorted by match score descending, score=0 hidden
- Card: name, city, "você recebe" / "você dá" sticker lists (max 12 + overflow)
- WhatsApp button: `wa.me/55{whatsapp}` if set, disabled otherwise

### `/u/[slug]` (server component, public)
- `notFound()` if slug invalid or `isPublic=false`
- Progress bar + stats
- Two columns: "Procuro" (missing) | "Tenho para troca" (duplicates), grouped by team
- Share button copies URL from headers() (works on any domain)
- `generateMetadata` for SEO

### `/perfil` (server + client form)
- Fields: displayName, city, whatsapp, isPublic toggle
- Shows public URL with copy button
- `updateProfile` server action with sonner feedback
